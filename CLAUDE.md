# CLAUDE.md ‚Äî Bittle MicroPython Project

## Project Overview

MicroPython control of a Petoi Bittle X V2 quadruped robot running on its BiBoard V1.0 (ESP32).
Replaces the stock OpenCat firmware with hand-written Python.

## Architecture

### Layer separation

```
src/drivers/servo.py        ‚Äî hardware only: PWM/GPIO, no calibration knowledge
src/poses.py                ‚Äî poses layer: calibration, motion, named poses
src/battery.py              ‚Äî battery voltage monitoring (GPIO 37, BiBoard V1.0)
src/gaits/walk.py           ‚Äî walk gait: 116-frame OpenCat keyframe sequence
src/demos/stand.py          ‚Äî demo: stand ‚Üí sit ‚Üí stand ‚Üí rest
src/demos/walk.py           ‚Äî demo: stand ‚Üí walk ‚Üí rest
src/server.py               ‚Äî HTTP command server (_thread + raw sockets, port 80)
src/boot.py                 ‚Äî WiFi connect + WebREPL start (runs on every boot)
src/main.py                 ‚Äî HTTP server loop (runs after boot.py)
src/webrepl_proxy.py        ‚Äî host-side PTY proxy: bridges mpremote ‚Üî WebREPL WebSocket
src/configuration/wifi_config_template.py ‚Äî credential template (copy ‚Üí wifi_config.py)
```

`servo.py` is a clean hardware abstraction. Calibration and robot-specific geometry belong to `poses.py`, not the driver.

### BiBoard V1 servo channels (Bittle)

```
CH 0  ‚Üí Head pan
CH 4  ‚Üí Front Left Shoulder   (rotDir +1)
CH 5  ‚Üí Front Right Shoulder  (rotDir -1)
CH 6  ‚Üí Rear Right Shoulder   (rotDir -1)
CH 7  ‚Üí Rear Left Shoulder    (rotDir +1)
CH 8  ‚Üí Front Left Leg        (rotDir -1)
CH 9  ‚Üí Front Right Leg       (rotDir +1)
CH 10 ‚Üí Rear Right Leg        (rotDir +1)
CH 11 ‚Üí Rear Left Leg         (rotDir -1)
Channels 1, 2, 3 unused.
```

GPIO pin mapping (channel ‚Üí GPIO): `[18, 5, 14, 27, 23, 4, 12, 33, 19, 15, 13, 32]`

### Servo PWM

`servo.py` runs at **200Hz** (default). The ESP32 LEDC peripheral is 10-bit at this frequency,
giving ~409 steps across the servo range (0.44¬∞/step). At 50Hz it is only 102 steps (1.76¬∞/step)
which produces visible stepping/jerkiness. Bittle's digital servos support up to ~330Hz.

### Calibration system

`config.py` stores per-channel offsets (degrees from 90¬∞). It is **gitignored** ‚Äî each robot has different offsets. Generate it by running `src/configuration/calibrate.py` in the REPL and copying the `done()` output.

`apply_calibration(angle, ch)` adds the offset before sending to hardware.
`_REST_COMMANDED` in `poses.py` uses *commanded* (pre-calibration) angles.
`current_pos` in `poses.py` tracks *calibrated* angles for smooth interpolation.

### Angle conventions

**For named poses** (`stand`, `sit`, `rest` in `poses.py`):
```
commanded_angle = 90 + rotationDirection[joint] * opencat_angle
```

**For gait keyframes** ported from OpenCat `InstinctBittleESP.h`:
```
commanded_angle = ZERO_POS[joint] + rotationDirection[joint] * opencat_raw
```
Where `ZERO_POS` accounts for `middleShift[]` in OpenCat (the mechanical neutral per joint):
```
FL_sh=65, FR_sh=115, RR_sh=115, RL_sh=65
FL_leg=80, FR_leg=100, RR_leg=100, RL_leg=80
```
Verified: OpenCat `rest[]` values produce exactly the same commanded angles as `_REST_COMMANDED`.

## Deployment with mpremote

Scripts run with `mpremote run` execute in device context ‚Äî imports resolve against the device filesystem.

### Stand demo
```bash
mpremote fs cp src/drivers/servo.py :servo.py + \
    fs cp src/poses.py :poses.py + \
    fs cp config.py :config.py + \
    run src/demos/stand.py
```

### Walk demo
```bash
mpremote fs cp src/drivers/servo.py :servo.py + \
    fs cp src/poses.py :poses.py + \
    fs cp config.py :config.py + \
    fs mkdir :gaits + \
    fs cp src/gaits/walk.py :gaits/walk.py + \
    run src/demos/walk.py
```

Note: `fs mkdir :gaits` will error if the directory already exists ‚Äî safe to ignore.

### WiFi setup (one-time USB upload)
```bash
mpremote fs cp src/drivers/servo.py :servo.py + \
    fs cp src/poses.py :poses.py + \
    fs cp src/battery.py :battery.py + \
    fs cp config.py :config.py + \
    fs cp wifi_config.py :wifi_config.py + \
    fs cp src/boot.py :boot.py + \
    fs cp src/server.py :server.py + \
    fs mkdir :gaits + \
    fs cp src/gaits/walk.py :gaits/walk.py + \
    fs cp src/main.py :main.py
```

### mpremote over WiFi (after WiFi setup)
```bash
# Terminal 1
python src/webrepl_proxy.py 192.168.1.x <password>
# Terminal 2 ‚Äî use the PTY path printed by the proxy
mpremote connect /dev/ttysNNN repl
mpremote connect /dev/ttysNNN run src/demos/walk.py
```
