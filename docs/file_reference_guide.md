# File Reference Guide

This document clarifies which files to use and how they're referenced in the documentation.

## Source Files vs Documentation

All code should be maintained in the source files, not duplicated in documentation. The getting started guide now references the actual files.

### Servo Driver: `drivers/pca9685.py`

**Purpose:** Complete PCA9685 servo controller driver for MicroPython

**Features:**
- Full 16-channel PWM control
- Servo angle control (0-180°)
- Direct pulse width control
- Error handling and validation
- Well-documented API

**Used by:** All other scripts that control servos

**Upload command:**
```bash
mpremote fs cp drivers/pca9685.py :pca9685.py
```

**Referenced in:**
- `MICROPYTHON_GETTING_STARTED.md` - Step 6
- `README.md` - Quick start

---

### Test Script: `test_servo.py`

**Purpose:** Test first servo movement and verify I2C connection

**What it does:**
1. Scans I2C bus for PCA9685 (0x40)
2. Initializes servo controller
3. Moves servo 0 through test positions
4. Verifies everything is working

**Run command:**
```bash
mpremote run test_servo.py
```

**Referenced in:**
- `MICROPYTHON_GETTING_STARTED.md` - Step 6
- `README.md` - Quick start

---

### Calibration Tool: `poses/calibrate.py`

**Purpose:** Interactive calibration tool to find neutral servo positions

**Features:**
- Interactive prompts for each of 9 servos
- Fine control (+/- 5° or 1° increments)
- Generates `config.py` with calibration offsets
- Named joints for easy identification
- Save/quit options

**Run command:**
```bash
mpremote run poses/calibrate.py
```

**Output:** Generates `config.py` (save to computer, then upload)

**Upload config:**
```bash
mpremote fs cp config.py :config.py
```

**Referenced in:**
- `MICROPYTHON_GETTING_STARTED.md` - Step 7
- `README.md` - Quick start

---

### Stand Demo: `poses/stand.py`

**Purpose:** Demonstrate standing, sitting, and resting poses

**Features:**
- Loads calibration from `config.py` (if available)
- Multiple poses: zero, stand, sit, rest
- Uses `set_calibrated_servo()` helper function
- Error handling (KeyboardInterrupt, exceptions)
- Graceful shutdown (returns to zero position)

**Run command:**
```bash
mpremote run poses/stand.py
```

**Requires:**
- `pca9685.py` uploaded to BiBoard
- `config.py` uploaded (optional, but recommended)

**Referenced in:**
- `MICROPYTHON_GETTING_STARTED.md` - Step 8
- `README.md` - Quick start

## Configuration File: `config.py`

**Purpose:** Store servo calibration offsets

**Auto-generated by:** `poses/calibrate.py`

**Format:**
```python
CALIBRATION = {
    0:  +0,  # Front Left Shoulder
    1:  -5,  # Front Left Leg
    2:  +2,  # Front Right Shoulder
    # ... etc
}

JOINTS = {
    'front_left_shoulder': 0,
    'front_left_leg': 1,
    # ... etc
}

def apply_calibration(angle, channel):
    return angle + CALIBRATION.get(channel, 0)
```

**Usage:**
```python
from config import CALIBRATION, apply_calibration

# Apply calibration when setting servo
calibrated_angle = apply_calibration(90, 0)
servos.set_servo(0, calibrated_angle)
```

## Development Workflow

### Recommended File Organization

**On BiBoard (uploaded):**
```
/
├── pca9685.py         # Servo driver
├── config.py          # Calibration data (after running calibrate.py)
└── main.py            # (Optional) Auto-run on boot
```

**On Computer (this repo):**
```
doggo/
├── drivers/
│   └── pca9685.py
├── poses/
│   ├── calibrate.py
│   └── stand.py
├── test_servo.py
└── config.py          # Generated by calibrate.py
```

### Upload Workflow

```bash
# 1. Upload driver (one time)
mpremote fs cp drivers/pca9685.py :pca9685.py

# 2. Calibrate servos (one time, or when needed)
mpremote run poses/calibrate.py
# ... follow prompts ...
# ... save config.py output to file ...

# 3. Upload calibration (after calibrating)
mpremote fs cp config.py :config.py

# 4. Run demos/tests (anytime)
mpremote run test_servo.py
mpremote run poses/stand.py

# 5. Develop custom code
# Edit files on computer, run with mpremote:
mpremote run my_custom_pose.py
```

## Key Differences: Old vs New Documentation

### Before (Duplicated Code)

❌ Getting started guide had embedded code for:
- Full PCA9685 driver (simplified version)
- Calibration function (basic version)
- Standing pose code (different from stand.py)

**Problems:**
- Code duplication between docs and source files
- Version drift (docs and source could become inconsistent)
- Maintenance burden (update code in multiple places)
- Confusion (which version to use?)

### After (File References)

✅ Getting started guide now:
- References actual source files
- Shows how to upload and run them
- Explains what each file does
- Shows expected output
- Provides small usage examples (not full code)

**Benefits:**
- Single source of truth (source files only)
- No version drift
- Easier maintenance
- Clear workflow
- Users get the best, most complete code
